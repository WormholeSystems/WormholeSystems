# Contributing

WormholeSystems is built in the open, and contributions of every size are welcome.

## The main repo

**[WormholeSystems/WormholeSystems](https://github.com/WormholeSystems/WormholeSystems)** — the main repository behind wormhole.systems, a Laravel (PHP) + Vue application.

Start by reading the repo's README, then:

- **Found a bug or have an idea?** Open an issue. It's the best place to discuss a change before building it.
- **Want to fix or build something?** Fork the repo, make your change, and open a pull request.

## Improving these docs

The documentation you're reading is the easiest place to start — every page is a plain Markdown file in the main repo. At the top of any page, **Copy as Markdown** gives you the source, and **Edit on GitHub** takes you straight to that file's editor on GitHub, where you can propose a change in a couple of clicks. Add a file and it shows up automatically — no code required.

## Why contribute

- **It helps everyone.** WormholeSystems is a shared, community tool — a fix or feature you add benefits every group that uses it.
- **It's yours too.** The same openness that lets you [self-host](/documentation/contributing-and-self-hosting/self-hosting) lets you shape where the project goes.

Want to talk through an idea first? [Join the Discord](/documentation/contributing-and-self-hosting/getting-help) — that's where development is discussed.
