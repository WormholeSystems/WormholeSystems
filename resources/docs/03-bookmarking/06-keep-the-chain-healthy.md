# Keep the chain healthy

The convention only pays off if everyone keeps to it. A few habits keep the chain accurate for the whole group.

## Post every signature you scan

It takes about ten seconds. The mapper even asks which signature you jumped through, so most of the work is already done for you ([prompt for signature](/documentation/bookmarking/let-the-mapper-name-it)).

Skipping it doesn't save time — it just **shifts the work to the next person** and slows everyone down. Don't be the reason someone else has to rescan a system that was already cleared.

## Keep bookmarks and the mapper in sync

Every resolved signature should exist in **both** places, named the same way. If a hole is on the map it should be in your bookmarks, and vice versa. When they drift apart, people stop trusting either one.

## Add connections to the shared map

All connections belong on the [shared map](/documentation/getting-started/overview) so routing, chain awareness, and activity tracking work for everyone — not just for you. A connection that lives only in your client's bookmarks is invisible to the rest of the group.

## The short version

- Bookmark every resolved signature **immediately**.
- Use the [agreed format](/documentation/bookmarking/why-this-system) — let the [mapper name it](/documentation/bookmarking/let-the-mapper-name-it).
- Keep bookmarks and the map matching.
- Post your scans so nobody has to redo them.
