# Right-click menus

The single most useful habit on the map: **right-click things.** Almost every element has a context menu, and that's where most actions live. If you're wondering how to do something, right-click whatever it relates to first.

Here's what each menu offers.

## Empty map background

- **Add Solarsystem** — search for and place a system.
- **Delete selection** / **Organize selection** — manage the systems you've selected.
- **Clear map** — remove all unpinned systems.

## A system

- **Pin / Unpin** — protect a system from cleanup.
- **Status** — mark it Unknown, Friendly, Hostile, Active, Unscanned, or Empty.
- **Set as Home System** and **Set as Rally Point** — the shared markers (see [Home & rally point](/documentation/navigation/home-and-rally-point)).
- **Remove** — delete the system (unless it's pinned or your home).
- **Set destination** / **Add waypoint** — push the system to your in-game autopilot, per character or for **All Characters**.
- **Route planner** — set the system as the route origin or destination.
- **External** — jump out to Dotlan or zKillboard for the system or region.

## A connection

- **Lifetime**, **Mass Status**, **Ship Size** — set the wormhole's [status](/documentation/connections/overview).
- **Copy name** — copy the correctly-formatted [bookmark](/documentation/bookmarking/let-the-mapper-name-it) for either end of the hole.
- **Remove connection** — delete the link.

## Characters, killmails, EVE Scout & skyhooks

These all share a menu for acting on the system they sit in: **Set destination**, **Add waypoint**, **Route planner**, **Add to map** (if it isn't already), **External** links, and **Set as Rally Point**.

## A note on permissions

Read-only actions — external links, setting destinations and waypoints, the route planner — are available to **everyone**, including viewers. Actions that change the shared map — adding, removing, status, pins, home, rally — need **Member** access or higher. See [Roles](/documentation/access-and-permissions/roles).
