# Roles

There are three access levels, plus the map owner. Each level includes everything below it.

## Viewer

Read-only. A Viewer can open the map and see systems, connections, and signatures — but **cannot see live character positions**, and can't change anything. Good for sharing intel with people you don't want editing the chain or tracking your fleet.

## Member

Everything a Viewer can do, **plus**:

- see **live character positions** on the map
- record jumps and build the chain
- add and remove systems, create and edit connections
- manage signatures, set aliases, status, and pins

This is the everyday mapping role for active members of your group.

## Manager

Everything a Member can do, **plus control of the map itself**:

- grant and revoke other people's access
- generate and revoke share links and other map settings

Only Managers (and the owner) can see and edit the [access list](/documentation/access-and-permissions/how-access-works).

## Owner

The map's creator. Has full Manager rights and is the **only** role that can **delete the map**. The owner's access can't be changed by anyone else.

## Quick comparison

| Action                                    | Viewer | Member | Manager | Owner |
| ----------------------------------------- | :----: | :----: | :-----: | :---: |
| View systems, connections, signatures     |   ✓    |   ✓    |    ✓    |   ✓   |
| See live character positions              |   —    |   ✓    |    ✓    |   ✓   |
| Edit the map (systems, signatures, chain) |   —    |   ✓    |    ✓    |   ✓   |
| Manage access & share links               |   —    |   —    |    ✓    |   ✓   |
| Delete the map                            |   —    |   —    |    —    |   ✓   |
